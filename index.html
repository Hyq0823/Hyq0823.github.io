核心类的理解和学习

SparkConf 类

SparkConf包含了初始化spark集群配置的参数，默认情况下，无参的构造函数：new SparkConf()，会加载spark.*开头的java系统变量，同时
也可以通过sparkConf.set(key,value)。的形式设置变量和值。
但是直接在对象上设置的同名属性，优先级高于系统变量，一旦创建，在运行时不支持动态修改。

SparkContext 类

代表着和sapark集群的链接。用于进行RDD的创建、累加和在集群中广播变量使用。

JavaSparkContext 类

SparkContext的Java版本，返回javardd并且使用Java集合而不是Scala集合。

JavaRDD 类

Spark是Scala写的。JavaRDD就是Spark为Java写的一套API。

JavaPairRDD 类

JavaRDD => JavaPairRDD: 通过mapToPair函数
JavaPairRDD => JavaRDD: 通过map函数转换

宽依赖和窄依赖

窄依赖：父RDD中，每个分区内的数据，都只会被子RDD中特定的分区所消费，为窄依赖：

宽依赖：父RDD中，分区内的数据，会被子RDD内多个分区消费，则为宽依赖：

一句话描述：父RDD中分区内的数据，是否在子类RDD中也完全处于一个分区，如果是，窄依赖，如果不是，宽依赖。

算子操作

spark中的数据都是抽象为RDD的，它支持两种类型的算子操作：Transformation和Action。Transformation算子的代码不会真正被执行。只有当我们的程序里面遇到一个action算子的时候，代码才会真正的被执行。

job和stage和task都是什么？

当在程序中遇到一个action算子的时候，就会提交一个job。

每个Job被分为小的计算任务组，每组称为一个stage。
而在一个job 中划分stage 的一个重要依据是否有shuflle 发生 ，也就是是否会发生数据的重组 （重新组织数据）。

shuffle依赖是两个stage的分界点。shuffle操作一般都是任务中最耗时耗资源的部分。因为数据可能存放在不同的节点上，

下一个stage的执行首先要去拉取上一个stage的数据（shuffle read操作），保存在自己的节点上，就会增加网络通信和IO。

而Task是真正执行的工作单元，对一个Stage之内的RDD进行串行操作的计算任务。多个Task组成一个Stage。

前言-CountDownLatch是什么？

CountDownLatch是具有synchronized机制的一个工具，目的是让一个或者多个线程等待，直到其他线程的一系列操作完成。

CountDownLatch初始化的时候，需要提供一个整形数字，数字代表着线程需要调用countDown()方法的次数，当计数为0时，线程才会继续执行await()方法后的其他内容。

Elasticsearch清理历史索引笔记

发表于 2020-02-24 分类于 Elasticsearch运维实战攻略 阅读次数： Changyan：
本文字数： 1.6k 阅读时长 ≈ 1 分钟
前言
  • es用作海量数据和存储和检索，对于较大日志数据，如果全部存储在一个索引中，
    像数据库表一样，久而久之，检索查询会越来越慢，服务器压力也大。无论是按天分索引、还是按月份索引，一般建立索引的时候都会带上时间。
  • 很明确的说，已经超过一定量级的索引，应该是不会再写入新数据了，可以强制进行合并压缩，设置只读等操作。
  • 对于搜索来说，合并的分片越少，查询和排序越快（想象一下深度分页。。。）
  • 我们的文档存储在分片中，并且在分片中被索引，但是我们的应用程序不会直接与它们通信，取而代之的是，直接与索引通信，像查询数据库表一样，我们可能只需要查询近两天的数据就能得到我们想要的结果，而不是扫描包含所有数据的大表。
    • 将数据按时间分索引存储，将所有数据全load到内存，查询不走磁盘，直接走内存是很快的。
